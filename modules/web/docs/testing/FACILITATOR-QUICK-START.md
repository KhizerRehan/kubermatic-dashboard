# Facilitator Quick Start Guide

Quick-reference guide for facilitators running the Testing Knowledge-Sharing Session.

**Target Audience:** Session facilitators
**Scope:** Session day checklist, key messages, troubleshooting
**Duration:** 5-10 minute read before session

---

## Day-Of Checklist (30 minutes before)

### Tech Setup (15 minutes)
- [ ] Laptop connected to projector/TV
- [ ] Presentation open and tested
- [ ] Zoom link working (if hybrid)
- [ ] Chat monitored (assign someone)
- [ ] Recording enabled (with consent)
- [ ] Microphone on and tested
- [ ] Browser has all links ready to share

### Room Setup (10 minutes)
- [ ] Feedback forms on table
- [ ] Sticky notes and markers ready
- [ ] Handouts stacked (agenda, resources)
- [ ] Snacks/beverages available
- [ ] Chairs arranged to see screen
- [ ] Feedback station ready (forms, QR code, sticky notes)
- [ ] Champion info sheets visible

### Facilitator Prep (5 minutes)
- [ ] Review slide notes
- [ ] Know where to go if tech fails
- [ ] Have backup printed slides
- [ ] Feedback survey link handy
- [ ] Champion contact list ready
- [ ] Timer ready (watch or phone)

## Opening (First 8 Minutes)

### Goals
- Welcome team warmly
- Set tone (collaborative, positive)
- Frame session purpose clearly
- Build excitement

### Script

```
Good [morning/afternoon]! Thanks for being here.

I'm [your name], and I'm excited to spend the next 90 minutes
with you talking about testing.

Before we start, quick logistics:
  • Session is 90 minutes (we'll try to stay on time)
  • Feedback survey at end (5-10 min, on device)
  • Questions? Ask anytime - just raise hand or type in chat
  • Restrooms are [location]

Why are we here today?

Over the last 6 months, we invested heavily in testing:
  ✓ Built testing utilities that make writing tests faster
  ✓ Created 15+ guides to help you succeed
  ✓ Improved test coverage across the codebase
  ✓ Established best practices and patterns

Today, we want to:
  1. Show you what we've built
  2. Help you find what you need
  3. Hear from you about what's working and what could improve
  4. Identify Testing Champions to help your teammates

This is YOUR time to ask questions, share feedback, and
let us know how we can support your success with testing.

Let's dive in!
```

## Handling Common Questions

### Q: "Why do we need all this documentation?"
**A:** "Good question! Documentation serves two purposes:
  1. Quick reference when you're writing tests
  2. Consistency so everyone approaches testing the same way

Think of it like a playbook - everyone knows the strategies,
so we're faster, higher quality, and aligned."

### Q: "My component is too complex to test"
**A:** "That's actually really valuable feedback! Complex components
usually point to design issues. Let's connect after this session
or in code review. Our Champions can help you break it down
or identify if refactoring would help testing."

### Q: "I never have time to write tests"
**A:** "I hear you - time pressure is real. A few things:
  1. Our utilities make tests faster (50% speedup on average)
  2. Champions can help you get unstuck
  3. Good tests save debugging time later

Let's talk one-on-one about your workflow and find
where testing can actually save you time."

### Q: "This seems like overkill for our team"
**A:** "Fair perspective! Here's why we've invested:
  • Testing prevents bugs that cost 10x more to fix
  • Our team has grown - consistency matters
  • Documentation helps new people onboard
  • Coverage actually catches real issues

But we hear you - let's make sure testing feels
right-sized for our workflow."

### Q: "Why CE/EE testing? We just build one thing"
**A:** "Great question! We build two editions, so:
  • Some features only exist in Enterprise
  • We need to test both versions
  • Coverage gaps in one edition create maintenance risk

TESTING-EDITIONS.md explains the whole system.
Champions can help make this concrete in code reviews."

## Managing Time

### Hard Stops (Don't Overrun)

**These are hard stops - keep on time:**
- 0:56 Q&A shift (move to discussion)
- 1:06 Testing Champions section (required)
- 1:24 Feedback collection (must happen)
- 1:34 Closing (important for next steps)

**Flexible time (can compress if behind):**
- Documentation tour (most info is written)
- Phase results (can show summary vs. detail)

**How to cut time if needed:**
- Skip some metrics breakdowns
- Refer people to specific docs
- Move detailed Q&A to office hours
- Shorten form test section (least critical)

### Keep Energy Up

- **Watch the room** - If people seem bored, move on
- **Vary cadence** - Don't stay on one topic >7 min
- **Use visuals** - Show screenshots, diagrams
- **Call on people** - "What does testing look like for your team?"
- **Celebrate** - "Great question! Others probably have that too"

## Facilitator Roles

### If You're the Presenter
- Maintain warm, energetic tone
- Make eye contact (even on Zoom)
- Use slides as anchor, not script
- Pause for questions
- Show genuine enthusiasm
- Manage time (watch clock)

### If You're the Feedback Collector
- Monitor form submissions
- Help anyone having trouble with survey
- Collect sticky note responses
- Watch for people raising hands
- Note interesting questions
- Watch chat (if hybrid)

### If You're the Note-Taker
- Capture key questions asked
- Note interesting comments
- Record feedback themes
- Track any "red flag" concerns
- Document action items mentioned
- Transcribe chat questions

### If You're the Time-Keeper
- Give 10-minute warning for transitions
- Prompt speaker for time checks
- Watch for over-runs
- Keep energy moving
- Help with bathroom breaks if needed

## Managing Difficult Moments

### Scenario: "This doesn't apply to my work"

**Response Options:**
1. **Acknowledge:** "That's fair feedback for your area"
2. **Reframe:** "Even if not today, knowing where to find answers helps"
3. **Offer Support:** "Let's talk after - there might be patterns that help"
4. **Move On:** Don't dwell - you can't make it perfect for everyone

### Scenario: "Testing slows us down"

**Response:**
1. **Agree partially:** "Initial test writing does take time"
2. **Offer counter:** "But fixes are 10x faster with tests"
3. **Share data:** "Our test suite catches bugs before production"
4. **Invite discussion:** "Tell me about your workflow - let's find gains"

### Scenario: Silent room / Low engagement

**Quick fixes:**
- Ask more open questions: "Who's written a test this week?"
- Use polls: "Show of hands - who uses mock utilities?"
- Small group chat: "Turn to a neighbor - what's your testing challenge?"
- Sticky notes: "Write one testing win this month" (low barrier)

### Scenario: Someone dominates Q&A

**Gracefully redirect:**
1. Thank them: "Great question, [Name]"
2. Offer one-on-one: "This is complex - let's chat after"
3. Note it: "We'll definitely address this in office hours"
4. Move on: "Other questions from folks who haven't asked yet?"

### Scenario: Tech fails (projector, Zoom, audio)

**Have backup plan:**
- Printed slides (for projector failure)
- Phone hotspot (for wifi failure)
- Print key content on poster (for all tech failure)
- Pivot to discussion (people, not slides matter)

**Keep calm:** "No worries, let's keep going. I'll share info verbally..."

## Feedback Collection Script

### Survey Explanation (When you show it)

```
"Time for your feedback! This takes 5-10 minutes.

We want to know:
  • What you found valuable
  • What confused you
  • Your biggest testing challenges
  • If you're interested in being a Testing Champion

→ QR code is in chat / here on screen
→ Or fill out paper form on the table

You can do this on your phone/laptop right now.
Champions will help if you need assistance."
```

### Getting Compliance

**Make it easy:**
- "Seriously, 5 minutes!"
- "I'll wait while you fill it out"
- Have helpers walk around
- Celebrate first responses: "Thanks for the feedback!"
- No pressure: "If you want to talk later, that's ok too"

**Offline option:**
- Have 10-15 printed forms
- "If you prefer paper, that's fine"
- Collect before people leave

### Sticky Note Collection

```
"One more quick thing - on your way out:
  • One sticky: Something that worked well
  • Another sticky: One thing to improve

Put them on the board in back.
Thanks for the feedback!"
```

## Closing (Last 6 Minutes)

### Key Messages to Reinforce

1. **Thank you** - "Thanks for caring about quality"
2. **Summary** - "You now know where to find everything"
3. **Champions** - "Experts ready to help"
4. **Office hours** - "Monthly meeting, next one [DATE]"
5. **Feedback** - "We're listening and improving"
6. **Questions** - "Email/Slack anytime"

### Closing Script

```
Thank you all for joining and for your feedback.

Here's what I want you to remember:

1️⃣  We've invested in making testing EASIER.
   Utilities, docs, examples - all here to help you.

2️⃣  You're not alone.
   Champions, office hours, #testing-help - support is ready.

3️⃣  Your feedback matters.
   We'll analyze what you said and improve based on it.

4️⃣  Testing is a team sport.
   We're better together.

What happens next:
  → Thank you emails (today)
  → Feedback analysis (this week)
  → Results published (2 weeks)
  → Champions reach out (2 weeks)
  → Monthly office hours continue (1st Thursday, 2-3 PM)

Questions? Stick around and grab me.

Let's make testing great! 🎉
```
